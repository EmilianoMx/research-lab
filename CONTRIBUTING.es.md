# Guía de contribución para investigación computacional

¡Gracias por considerar contribuir a este proyecto de investigación computacional! Este documento describe lineamientos y estándares para colaborar de forma efectiva.

## Revisión metodológica
- **Principios de ciencia abierta:** Comparte metodologías, datos y materiales siempre que sea posible y éticamente viable.
- **Revisión por pares interna:** Las nuevas metodologías deben ser revisadas por al menos una persona colaboradora antes de su adopción.

## Estándares de código
- **Estilo de código:** Sigue convenciones de nombres, indentación y comentarios consistentes.
- **Documentación:** Incluye documentación clara y concisa, con docstrings cuando corresponda.
- **Control de versiones:** Haz commits frecuentes con mensajes descriptivos.

## Gestión de datos
- **Organización de archivos:** Usa estructuras de carpetas lógicas y nombres descriptivos.
- **Compartición de datos:** Garantiza acceso para colaboradores según criterios éticos y regulatorios.
- **Formatos de datos:** Prioriza formatos estandarizados para facilitar interoperabilidad.

## Requisitos de reproducibilidad
- **Reproducibilidad:** Asegura que otros puedan ejecutar el trabajo y reproducir resultados.
- **Especificación de entorno:** Incluye archivos de dependencias y versiones (por ejemplo, `requirements.txt` o `environment.yml`).

## Estándares de salida
- **Reporte de resultados:** Presenta resultados de forma clara, con figuras y tablas correctamente etiquetadas.
- **Control de calidad:** Implementa validaciones para detectar errores de forma temprana.

## Guías de publicación
- **Autoría:** Define roles de autoría y responsabilidades desde etapas tempranas.
- **Citación de fuentes:** Reconoce y cita todas las fuentes y metodologías utilizadas.
- **Preprints y publicaciones:** Sigue el flujo definido por el equipo para envíos y difusión.

Gracias por contribuir a un ecosistema de investigación riguroso, colaborativo y transparente.
