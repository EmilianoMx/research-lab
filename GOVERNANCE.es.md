# Modelo de Gobernanza

## Alcance
Este repositorio define la gobernanza de políticas y estándares que guían la calidad científica, la reproducibilidad y la gestión responsable de datos en el Laboratorio de Investigación Computacional.

## Roles y responsabilidades
- **Mantenedor(es):** Definen la hoja de ruta, aprueban cambios estratégicos y curan estándares.
- **Contribuidores:** Proponen mejoras mediante pull requests y siguen las guías del proyecto.
- **Revisores:** Ejecutan controles de calidad sobre rigor metodológico, reproducibilidad y claridad.

## Toma de decisiones
- Los cambios se proponen mediante pull requests.
- Se requiere al menos una revisión para cambios de políticas.
- Las actualizaciones de alto impacto (gobernanza, ética, datos) deben incluir justificación y referencias.

## Criterios de calidad para merge
Todo pull request debe incluir:
1. Resumen claro de cambios y motivación.
2. Documentación actualizada cuando aplique.
3. Alineación con principios de reproducibilidad y datos FAIR.

## Versionado y trazabilidad
- La gobernanza y los estándares se versionan en el historial Git.
- Los cambios de política relevantes deben reflejarse en notas de release o en la descripción del PR.
